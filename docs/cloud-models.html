Imint Engine · Atmosfär · Modell-jämförelse

Cloud-detection-modeller — Kiruna 2024

Fyra Sentinel-2-scener över Kirunavaara & tundran, valda att täcka hela molnskalan från klart till täckande · samma pixlar genom fyra modellfamiljer

Modeller som jämförs

Sen2Cor SCL

ESA officiell · regelbaserad scen-klassificering · 12 klasser (klart, vegetation, vatten, snö, moln-låg/medium/hög, cirrus, skugga, …) · inbakad i alla L2A-produkter.

s2cloudless

Sinergise / Sentinel-Hub · LightGBM pixel-klassificerare · 10 TOA-band · returnerar [0, 1] cloud probability · github · MIT.

OmniCloudMask

DPIRD 2024 · transformer-segmenterare · 4 klasser: klart / tjockt / tunt / skugga · använder bara R/G/NIR/SWIR · github.

DES MLP5 (Pirinen 2024)

Digital Earth Sweden · 5-lagers MLP-ensemble (×3) · kontinuerlig Cloud Optical Thickness-regression · 11 band · github.

AOI & scen-urval

AOI: ~12×8 km centrerat på Kirunavaara (20.10–20.36°E, 67.81–67.89°N) — Kirunamalmen, samhället och tundran runtomkring. Period: 2024-04-01 → 2024-09-30. Scenerna valdes via SCL-stack-screening: fyra datum från olika kvartiler av AOI-cloud-fraction för att stresstesta modellerna mot både klart, snö-på-mark, partiellt moln och täckande.

Scen för scen

Varje scen visas med fem paneler i ordning: RGB · Sen2Cor SCL · s2cloudless (sannolikhet 0–1) · OmniCloudMask (4-klass) · DES MLP5 (kontinuerlig COT 0–0.3). Heatmaps använder samma färgskala där det är meningsfullt (grönt = klart, rött/vitt = moln).

s2cloudless och OmniCloudMask kör i en separat container (Python 3.11 + torch CPU). DES MLP5 och Sen2Cor-parsing körs på host. Tiderna inkluderar bara modell-inferens — inte spektral-fetch.

Reproducera

docker build -t imint/cloud-models:latest docker/cloud-models
python demos/cloud_models/fetch_kiruna.py
python demos/cloud_models/run_comparison.py

Containern hostar omnicloudmask + s2cloudless (kräver Python 3.10+). Host-koden hanterar DES MLP5 och Sen2Cor SCL-parsing.
